Architecture Report: fafsa-engine

Date: 2026-05-12

Do not implement yet. This report is read-only analysis. Pre-existing git state: M .gitignore.

Reviewed README, structure, FAFSA calculation Modules, ISIR validation, wizard/app layers, and tests. No ADR conflicts were found. Vocabulary: Module, Interface, Implementation, Depth, Shallow, Seam, Adapter, Leverage, Locality.

1. Formula Engine Module

Files: fafsa/kb.py, tests/test_fafsa_kb.py, fafsa/validate.py

Problem: Inputs, ED tables, formulas, citations, and trace construction share one broad Interface in fafsa/kb.py.

Solution: Keep prove_sai() public, but deepen a Formula Engine seam that composes table Modules and line-calculation Modules into a trace.

Benefits: Better Locality for ED formula defects and more Leverage from the stable prove_sai Interface.

Test Impact: Line-level tests plus end-to-end trace tests.

Risk: Moderate because ED validation is currently red.

Suggested Validation: python -m pytest tests/test_fafsa_kb.py tests/test_isir_validation.py -v

2. ISIR Reconstruction Adapter

Files: fafsa/isir.py, fafsa/validate.py

Problem: Fixed-width parsing, reconstruction heuristics, and comparison reporting are coupled, making the red gate hard to diagnose.

Solution: Split parser, reconstruction, and comparison Modules behind validate_isir_file().

Benefits: Better Locality around validation failures and clearer Leverage from ED records as a test oracle.

Test Impact: Focused fixed-width and reconstruction tests.

Risk: Low to moderate; preserve current red baseline until correctness work is assigned.

Suggested Validation: python -m pytest tests/test_isir_validation.py -v

3. Fact Intake And Trace Presentation Adapters

Files: demo.py, app.py, fafsa/wizard.py, llm/

Problem: Frontends and LLM backends know overlapping fact and trace presentation details.

Solution: Add narrow fact-intake and trace-presentation Adapter Interfaces around the deterministic engine.

Benefits: Better Locality for UX and backend changes; more Leverage for all frontends.

Test Impact: Deterministic fact normalization tests plus existing mocked backend tests.

Risk: Low.

Suggested Validation: python -m pytest tests/test_llm_backends.py tests/test_fafsa_kb.py -v
